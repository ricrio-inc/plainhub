# PlainHub ユーザーガイド — GitHub ファイルをブラウザで編集

PlainHubは、GitHubリポジトリのファイルをブラウザで直接編集できるAI対応オンラインエディタです。

---

## PlainHub とは

PlainHub は、GitHub リポジトリのファイルをブラウザで直接編集できるオンラインテキストエディタです。

- **完全なデータ所有権（Complete Data Ownership）** — ファイルはすべて GitHub リポジトリに保存。PlainHub 独自のサーバや DB は不要
- **100% クライアントサイド** — ブラウザ上で完結。PlainHub サーバにデータは送信されない
- **自動バージョン管理** — 保存のたびに Git コミットが自動作成。変更履歴は GitHub で確認可能
- **マルチデバイス** — どのデバイスからでもサインインしてアクセス可能
- **軽量** — 数万行の超ロングファイルでも入力が重くならない

### 対象ユーザー

- エンジニア、非エンジニア職種を問わず GitHub と AI を文書管理に活用したい人
- 手軽にブラウザからファイルを編集・保存したい人
- AI ツール（Claude Code / Cursor）から GitHub ファイルを操作したい人

## はじめかた

### ブラウザで使う

1. [plainhub.dev](https://plainhub.dev) にアクセス
2. **Sign in with GitHub** をクリック
3. リポジトリを選択してファイルを編集

### CLI で使う

```bash
npm install -g plainhub
plainhub auth --from-gh
plainhub open README.md -r your/repo
```

### MCP Server で使う（Claude Code）

`~/.claude/settings.json` に追加:

```json
{
  "mcpServers": {
    "plainhub": {
      "command": "npx",
      "args": ["-y", "plainhub-mcp"]
    }
  }
}
```

## エディタ機能

### 3つのエディタモード

| モード | 説明 | 用途 |
|--------|------|------|
| **Code** | プレーンテキスト/ソースコード表示（CodeMirror） | コード編集、テキスト編集 |
| **Preview** | Markdown レンダリング表示 | ドキュメント閲覧 |
| **Visual** | WYSIWYG 編集（ProseMirror） | Markdown を知らなくても OK。Word 感覚で編集 |

### 画像貼り付け

Ctrl+V でクリップボードのスクリーンショットを直接貼り付け可能。エビデンス付きドキュメントの作成に便利。

### キーボードショートカット

| ショートカット | 機能 |
|---|---|
| Ctrl+S | 即時保存（Git コミット） |
| Ctrl+B | サイドバー表示/非表示 |
| Ctrl+H | Focus Mode（UI 要素を非表示にして集中） |
| Ctrl+Z / Ctrl+Y | アンドゥ / リドゥ |
| Ctrl+L | 行選択 |
| Ctrl+Home / End | ファイルの先頭/末尾へ移動 |
| Alt+Z | 行の折り返し切り替え |

## AI パネル

エディタに統合された AI チャット機能。Claude API を使用し、自然言語でエディタを操作できます。

### できること

```
コンテンツ操作:
  「この段落を箇条書きに変換して」
  「このファイルの内容を要約して」
  「英語に翻訳して」
  「READMEを新規作成して、プロジェクトの概要を書いて」

UI操作:
  「ダークモードにして」
  「フォントサイズを大きくして」
  「サイドバーを閉じて」
  「プレビューモードに切り替えて」
```

### GitHub の操作を自然言語で

GitHub 特有の操作を AI パネルから自然言語で実行できます。GitHub の UI を覚える必要がありません。

| GitHub の操作 | PlainHub なら |
|---|---|
| リポジトリページを開いてファイルを探す | 「README.md を開いて」 |
| リポジトリ間を移動する | 「design-docs に切り替えて」 |
| 新しいリポジトリを作成する | 「my-notes というリポジトリを作って」 |
| ファイルを新規作成してコミットする | 「docs/memo.md を作成して」 |
| Issue を起票する | 「バグ報告の Issue を作成して」 |
| Issue にコメントする | 「Issue #1 に対応完了とコメントして」 |
| Issue をクローズする | 「Issue #1 を閉じて」 |

### 選択可能なモデル

| モデル | 特徴 |
|--------|------|
| **Haiku**（デフォルト） | 高速・低コスト。日常的なメモ編集や簡単な質問に最適 |
| **Sonnet** | バランス型。コード編集や複雑な提案に最適 |
| **Opus** | 最高性能。高度な分析や長文の構成に最適 |

設定画面からワンクリックで切り替え可能。

### 特徴

- **BYOK**（Bring Your Own Key） — 自分の Anthropic API キーを使用
- 編集中のファイル内容をコンテキストとして自動提供
- 提案編集の diff 表示で変更内容を確認してから適用
- ストリーミングレスポンス対応

## 検索機能

### 3つの検索モード

| モード | 説明 |
|--------|------|
| **Name** | ファイル名で検索 |
| **Content** | ファイルの内容から検索 |
| **All repos** | 全リポジトリ横断検索 |

### リポジトリ横断検索（All repos）

GitHub 上に分散しているファイルを、リポジトリの壁を越えて一括検索できる機能。

- **対象**: ログインユーザーがアクセス可能な全リポジトリ
- **検索方式**: ファイル名とコンテンツを同時検索（GitHub Search API + Trees API）
- **結果表示**: リポジトリごとにグループ化
- **絞り込み**: リポジトリ別フィルターで特定リポジトリの結果だけ表示可能

#### 活用例

- 複数リポジトリに散在するドキュメントから目的の情報を横断的に探す
- プロジェクト横断で特定のキーワード（API 名、設定値など）を検索
- 「あのファイルどのリポジトリにあったっけ？」を即座に解決

## リポジトリ管理

- **リポジトリ切り替え**: サイドバーヘッダーからワンクリックで切り替え
- **リポジトリ作成**: ブラウザ上から新規リポジトリを作成（Public/Private 選択可）
- **ファイル作成・編集**: ブラウザ上で直接ファイルの作成・編集・保存
- **コンテキストメニュー**: ファイルの複製、パスコピー、削除など

---

## セキュリティ

- **認証**:
  - Web UI: GitHub 公式の OAuth フローで安全にサインイン（トークンはブラウザの localStorage に保存）
  - CLI / MCP Server: `~/.config/plainhub/token` にローカル保存（パーミッション 0600、所有者のみ読み書き可）
- **コード整合性検証**: 実行中のコードが GitHub リポジトリと一致するか SHA-256 ハッシュで検証可能
