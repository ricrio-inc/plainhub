# PlainHub ユースケース — ドキュメント管理をGitHubで

**エンジニアもビジネスも、同じ場所で。**

GitHubでドキュメント管理・バージョン管理を行いたいチームのためのユースケース集。AI駆動オンラインエディタ、GitHubがバックエンド。

---

## こんなとき、PlainHub

| こんなとき | 課題 | PlainHub なら |
|-----------|------|-------------|
| **ドキュメントが散らかってきた** | どれが最新かわからない、管理ツールが乱立 | GitHub に一元化。履歴も自動記録。AI に「探して」と頼める |
| **データを自分で管理したい** | 特定サービスに依存したくない | Git 標準形式で保存。データは常に GitHub に。いつでも移行可能 |
| **外出先で修正したい** | PC がない、IDE がない | ブラウザだけで編集・保存。スマホからでもアクセス可能 |
| **AI と一緒に作業したい** | AI ツールとドキュメントが分断されている | AI パネル内蔵。編集しながら AI に相談できる |
| **大きなファイルを編集したい** | 他のエディタだと数千行で重くなる | 数万行の超ロングファイルでもサクサク編集可能 |

## ドキュメント履歴・復元を管理したい

多くのチームが抱える「あのファイル、前の状態に戻したい」「誰がいつ変更した？」という悩み。

PlainHub は保存のたびに Git コミットが自動作成されるため、**追加操作なしで**以下が実現します：

- **変更履歴の自動記録** — 全ファイルの変更が Git に記録される。「いつ・誰が・何を変えたか」が常に追跡可能
- **任意の時点への復元** — Git 履歴から過去の状態を参照・復元可能
- **差分表示（diff）** — 変更前後の差分をビジュアルに確認
- **ブランチ不要** — 非エンジニアでも main ブランチに直接保存するだけでバージョン管理が完了

> 一般的なクラウドサービスにも変更履歴はあるが、Git は分散型バージョン管理システムであり、堅牢で長期間の履歴を保持できる。

## 職種別の活用シーン

### エンジニア

- コードレビュー中にドキュメントをブラウザで確認・修正
- Claude Code で分析 → PlainHub でレポート作成 → 同じリポにコミット
- `plainhub diff` で変更差分をビジュアル確認
- MCP Server 経由で AI IDE から自然言語でファイル操作

### PM / 事業企画

- エンジニアと同じ GitHub リポでドキュメントを管理
- 議事録を作成 → AI に「TODO を Issue にして」と依頼
- 企画書を書きながら AI に壁打ち「このアイデアの弱点は？」

### 非エンジニア

- Markdown を知らなくても Visual モード（WYSIWYG）で Word 感覚で編集
- GitHub の複雑な操作を覚える必要なし — AI パネルに話しかけるだけ
- 画像もクリップボードから Ctrl+V で貼り付け

### チーム利用

- 同じリポジトリで全員がドキュメントを共有
- 変更履歴は Git が自動記録 — 「誰がいつ何を変えたか」が明確
- Deep linking で設定付き URL を共有 — チーム全員が同じ表示で確認

## github.dev との違い

github.dev はフル機能のエディタ — 本格的なコーディングセッションに最適。

PlainHub は **GitHub をバックエンドにしたメモ帳** — 開いて、編集して、保存。用途が異なるツールです。
