# PlainHub の特徴 — AI対応 GitHub オンラインエディタ

**Frontend for GitHub.**

GitHubをバックエンドにしたAI Nativeなオンラインエディタ。非エンジニアでもGitHubのファイルを簡単に編集できます。

---

## AI Native なオンラインエディタ

PlainHub はエディタに AI が統合されている。MCP/CLI 連携により、Claude Code や Cursor からも操作可能。

```
「この段落をもっと簡潔にして」 → AIが編集提案 → diff確認 → ワンクリックで適用
```

---

## リポジトリ全体と対話する — AIがリポを読んで一緒に考える

開いているファイルだけでなく、**AIはリポジトリ全体を読んで答える**。コード、ドキュメント、コミット履歴 — どれもAIから参照可能。Claude Codeのような体験を、ブラウザの中で。

```
「このリポでは認証ってどう実装してる？」
  → AIが関連ファイルを探して読み、構成と方針を要約

「先週何を作業した？」
  → AIが直近のコミットを確認し、変更点をまとめる

「app/index.html はどう変わってきた？」
  → AIが履歴をたどり、変更の流れを説明

「最近更新したファイル5つ教えて」
  → AIがコミットから抽出して提示
```

目の前のファイルをAIと編集するだけでなく、リポジトリ全体を相棒のAIと探索できる。

**ブレストして、そのまま残す**

リポ全体の文脈を持ったAIとアイデアを壁打ちし、結論をMarkdownファイルとして保存。会話はGitに残り、後日ファイルを開けば**そこから議論を再開できる**。流れて消えるチャットではなく、**蓄積される思考の場**になる。

```
1. 新しいメモを開く（例: ideas/new-feature.md）
2. AIに相談 — AIはリポ全体を読んで文脈を踏まえた提案
   「このリポの既存機能を踏まえて、新機能Xの設計を一緒に考えて」
3. 議論の要点をAIにまとめさせ、ファイルに保存
4. 翌日同じファイルを開く → AIが過去の議論を読んで続きから対話
```

## 声で指示、声で聞く — 入力も出力も音声に対応

AIパネルにPush-to-talk音声入力と、AI応答の音声読み上げ（TTS）を搭載。**話しかけて、聞き返す** — 移動中・作業中でも、画面に張り付かずにAIと対話できる。

**Phase 1（実装済み）**: Push-to-talk + Web Speech API + TTS
- マイクボタン長押し → 話す → 離すと自動送信 → AI が実行
- AI応答の音声読み上げ（TTS）— 1x〜5x の速度切替で「ながら聞き」可能
- ブラウザ標準 API（Web Speech API）使用、追加コスト 0
- Chrome/Edge/Safari 対応
- モバイル（タッチ）対応

**Phase 2（Coming Soon）**: 高精度音声認識
- Whisper API（BYOK）対応 — 技術用語の高精度認識

**Phase 3（Coming Soon）**: 会話モード（完全ハンズフリー対話）
- 連続会話モード（AI 応答後に自動で次の音声入力待ち、ボタン操作不要）
- ウェイクワード対応で運転中・料理中でも完全に手を離して対話

```
声で話す: 「さっきの会議の議事録をまとめて、TODOをIssueにして」

PlainHub が実行:
  1. 音声 → テキスト化
  2. AI パネルが意図を理解
  3. 議事録ファイルを作成
  4. TODO を GitHub Issue に起票
  5. すべて Git コミット
```

## どこからでも修正・デプロイ — 保存するだけで本番反映

PlainHub で保存 = GitHub にコミット。Vercel、Netlify、Cloudflare Pages など GitHub 連携済みのホスティングサービスを使えば、**保存した瞬間に自動デプロイ**が走る。

```
外出先でPlainHubを開く
  → ドキュメントの修正、コンテンツ更新
  → AIに「この文章をもっと簡潔にして」と相談
  → 保存（Ctrl+S）
  → GitHubにコミット → 自動デプロイ → 本番サイトに反映

PCもIDEもターミナルも不要。ブラウザだけで完結。
```

## アイデア開発 — 書きながら AI と壁打ち

アイデアを書き溜めるだけでなく、AI と対話しながらアイデアを発展させる。PlainHub ならメモと AI が同じ画面にあり、**AIはリポ全体の文脈を踏まえて提案**してくれる（→「リポジトリ全体と対話する」参照）。

```
1. アイデアをメモする（例: 新サービスの企画、記事の構想、事業計画）
2. AIに相談する
   「このアイデアの弱点は？」
   「競合との差別化ポイントを3つ考えて」
3. AIの提案を取り込んで、その場でメモを更新
4. 保存 → Git履歴にアイデアの発展過程が自動記録
```

他のツールとの違い:
- ChatGPT や Claude との会話 → **会話は流れていく。結果をどこかに書き写す必要がある**
- PlainHub → **アイデアの文書そのものが AI との作業場。編集結果がそのまま残る**

## GitHub Issues 連携

ドキュメントを編集しながら Issue の確認・作成・コメント・クローズがすべて AI パネルから可能。GitHub の Issues 画面を開く必要がない。GitHub の課題管理を AI から作成・管理できる。

```
「このリポジトリのIssue一覧を見せて」
「バグ報告のIssueを作成して」
「Issue #1 の内容を教えて」
「Issue #1 に対応完了とコメントして」
「Issue #1 を閉じて」
```

## データ主権（Data Ownership）

PlainHub の最も重要な設計原則。データは 100% ユーザーの GitHub リポジトリに帰属する。

### 原則

- **PlainHub はデータを一切保持しない** — 独自のサーバ・DB なし
- **データの所在は GitHub** — ファイルはすべてユーザーの GitHub リポジトリに保存
- **PlainHub をやめてもデータは残る** — Git 標準形式なので他サービスへの移行も容易
- **100% クライアントサイド** — ブラウザと GitHub 間の直接通信のみ

### 他サービスとの違い

| サービス | データの所在 | サービス停止時 |
|---------|-----------|-------------|
| 一般的なクラウドサービス | サービス提供元のサーバ | エクスポートが必要 |
| **PlainHub** | **ユーザーの GitHub** | **データはそのまま残る** |

### なぜ重要か

- ベンダーロックインがない
- データの可搬性（Git 標準形式）
- 監査・コンプライアンスに対応しやすい（Git 履歴で全変更を追跡可能）

## エンタープライズ

### 導入ステップ

| Phase | 対象 | 内容 |
|-------|------|------|
| **Free** | 個人・小規模チーム | 現在の PlainHub 機能をそのまま利用（Public/Private リポ対応） |
| **Team** | 部門・プロジェクト単位 | GitHub Organization 連携、メンバー管理、共有リポジトリ |
| **Enterprise** | 全社導入 | SSO/SAML 連携、MFA、監査ログ、IP 制限、カスタムドメイン、セルフホスト、SLA |

### エンタープライズで求められる機能

- **GitHub Enterprise Server / Cloud 対応** — オンプレミス GitHub との接続
- **SSO (SAML/OIDC)** — 企業の既存認証基盤と連携
- **監査ログ** — 誰がいつどのファイルにアクセス・編集したかの記録
- **IP 制限 / アクセス制御** — 社内ネットワークからのみアクセス可能に
- **セルフホスト対応** — 自社インフラ上で PlainHub を運用可能に
- **カスタムブランディング** — 企業ロゴ、カスタムドメイン

### 訴求ポイント

- **高いデータポータビリティ** — データは Git 標準形式で GitHub に残る。PlainHub をやめてもデータは失われない
- **既存インフラ活用** — GitHub Enterprise を既に使っている企業はすぐに導入可能
- **学習コスト最小** — 非エンジニアでもブラウザで開いて書くだけ
